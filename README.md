### Java性能优化

<pre>
性能优化方法论
</pre>

<pre>
1 更少的代码
    1）小程序运行比大程序要快
    2）需要编译的代码越多等待程序启动耗时越长
    3）需要创建的对象越多，垃圾收集量就越大，要分配持有的对象越多，GC周期就越长
    4）从磁盘装载进入JVM的类越多，程序启动话费时间越长
    5）要执行的代码越多，机器硬件缓存的效率就越低，耗时会越长
    随着单个应用程序的功能越来越大，程序的运行效率越来越低，要避免积少成多的效率降低，适时地的拆分应用
2 更优秀的算法
3 数据库的瓶颈
</pre>
